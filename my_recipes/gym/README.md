# Qwen Gym -> verl Train Scripts

## 1) Convert qwen_gym JSONL to verl parquet

```bash
cd /mnt/code/yehangcheng/github/verl

python3 train_scripts/prepare_qwen_gym_parquet.py \
  --train_jsonl /mnt/code/yehangcheng/ms-swift/qwen_gym/dapo_math_17k_swift_nemo_gym.jsonl \
  --val_jsonl /mnt/code/yehangcheng/ms-swift/qwen_gym/aime_2024_swift_nemo_gym.jsonl \
  --output_dir /mnt/code/yehangcheng/github/verl/train_scripts/data/qwen_gym_math \
  --verify_url http://127.0.0.1:18001/verify
```

This generates:

- `train_scripts/data/qwen_gym_math/train.parquet`
- `train_scripts/data/qwen_gym_math/val.parquet`

## 2) Start your Gym verifier service

Make sure your `/verify` server is already running, for example:

- `http://127.0.0.1:18001/verify`

## 3) Run verl training

```bash
cd /mnt/code/yehangcheng/github/verl

MODEL_PATH=Qwen/Qwen2.5-3B-Instruct \
TRAIN_FILES=/mnt/code/yehangcheng/github/verl/train_scripts/data/qwen_gym_math/train.parquet \
VAL_FILES=/mnt/code/yehangcheng/github/verl/train_scripts/data/qwen_gym_math/val.parquet \
N_GPUS_PER_NODE=8 \
NNODES=1 \
bash train_scripts/run_qwen_gym_math.sh
```

## Optional overrides

You can override these variables:

- `TRAIN_BATCH_SIZE`
- `PPO_MINI_BATCH_SIZE`
- `PPO_MICRO_BATCH_SIZE_PER_GPU`
- `ROLLOUT_N`
- `MAX_PROMPT_LENGTH`
- `MAX_RESPONSE_LENGTH`
- `PROJECT_NAME`
- `EXPERIMENT_NAME`

## Notes

- `train_scripts/interaction_config/gym_interaction_config.yaml` uses `GymInteraction`.
- Each sample carries `extra_info.interaction_kwargs` with `name: "gym"` and `env_config` generated by converter.
- Reward is returned by Gym verifier and written to `rm_scores` for PPO/GRPO training.
